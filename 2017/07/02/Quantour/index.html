Maven整合的Spring,hibernate,Struts2,Ajax,json的Web项目

软工三的大作业是要做一个股票的系统，前两个迭代周期是做一个桌面系统，第三个迭代周期要求转到web项目。这是第一次尝试完整的web项目，尽管假期提前看过一些JSP，Servlet，hibernate框架和Struts2框架也做过一些小Demo。但发现实战起来还是遇见了很多未知的困难。在这里记录一下这学期完整的开发过程，尽管我们组最终的成品质量不高，也没有很好的达到自己的预期，不过在每个新事物的尝试中在作品加入新技术都带给自己很多的提升。

1. Maven建立Web项目

  • 创建一个maven的project，勾选上Create a simple project （不使用骨架）

  • 这里的Packing 选择 war的形式
  • 由于packing是war包，那么下面也就多出了webapp的目录
  • 我们的项目要使用eclipse发布到tomcat下面，这里我们需要先把项目转成dynamic web project。在我们的项目上点击右键，选择properties 并找到Project Facets，并点击Convert to faceted form…然后勾选Dynamic Web Module并点击ok如下图：(3.0只有tomcat7才支持)
  • 接下来观察我们的项目结构，多了一个web content目录
  • 虽然此时我们可以发布到tomcat中，但这不符合maven的结构，我们还要做如下修改：把上图WebContent下面两个目录 META-INF ，WEB-INF 直接剪切到src/main/webapp目录下，并删掉WebContent目录，那么现在的项目结构如下图：
  • 然后我们要修改发布规则，右键点击项目，选择Deployment Assembly。选择WebContent，把它remove掉。
  • 接着重新指定一个web的路径，点击add，选择Folder->next。在src下找到webapp目录，然后finish。

  • 最后一步，我们要把当前的build path指向 Maven Dependency， 直接点击add，选择Java Build Path Entries 然后next。选择Maven dependencies。
  • 大家发现前几步一直有报红，最后需要在WEB-INF下面添加web.xml配置文件，此时pom不报错了，java Resource还是有红，我此时将打包方法从war改成jar，不报错了。开始选war也许只是想出现webapp文件夹，其实你也可以手动添加。此时web项目建立完成。
