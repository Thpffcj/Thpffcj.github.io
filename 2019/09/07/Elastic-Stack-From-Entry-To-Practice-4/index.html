Logstash 入门

1. Logstash 篇之入门与运行机制

1. 入门及架构简介

  • 数据收集处理引擎
  • ETL工具

架构简介

  • Pipeline
    • input-filter-output的3阶段处理流程
    • 队列管理
    • 插件生命周期管理
  • Logstash Event
    • 内部流传的数据表现形式
    • 原始数据在input被转换为Event，在output event被转换为目标格式数据
    • 在配置文件中可以对Event中的属性进行增删改查

Thpffcj:logstash-6.5.4 thpffcj$ cat config/codec.conf 
input {
  stdin {
    codec => line
  }
}

output {
  stdout {
    codec => json
  }
}

Thpffcj:logstash-6.5.4 thpffcj$ echo "foo
> bar
> " | bin/logstash -f config/codec.conf
> 
>

2. Life_of_an_Event

3. queue简介

Queue的分类

  • In Memory
    • 无法处理进程Crash，机器宕机等情况，回导致数据流失
  • Persistent Queue In Disk
    • 可处理进程Crash等情况，保证数据不丢失
    • 保证数据至少消费一次
    • 充当缓冲区，可以替代Kafka等消息队列的作用

PQ的基本配置

  • queue.type.persisted
    • 默认是memory
  • queue.max_bytes
    • 队列存储最大数据量，默认是1gb

4. 线程简介

  • pipeline.workers | -w
    • pipeline线程数，即filter_output的处理线程数，默认是cpu核数
  • pipeline.batch.size | -b
    • Batcher一次批量获取的待处理文档数，默认125，可以根据输出进行调整，越大会占用越多的heap空间，可以通过jvm.options调整
  • pipeline.batch.delay | -u
    • Batcher等待的时长，单位为ms

5. 配置简介

  • logstash设置相关的配置文件(在conf文件夹中，setting files)
    • logstash.yml：logstash相关的配置，比如node.name，path.data，pipeline.wokers，queue.type等，这其中的配置可以被命令行参数中的相关配置覆盖
    • jvm.options修改jvm的相关参数，比如修改heap size等
  • pipeline配置文件
    • 定义数据处理流程的文件，以.conf结尾

配置方式建议

  • 线上环境推荐采用配置文件来设定logstash的相关配置，这样可以减少犯错的机会，而且文件便于进行版本话管理
  • 命令行形式多用来进行快速的配置测试，验证，检查等

6. 多实例运行

  • bin/logstash –path.settings instance1
  • bin/logstash –path.settings instance2
  • 不同instance中修改logshtash.yml，自定义path.data，确保其不相同即可

7. pipeline配置简介

  • 用于配置input，filter和output插件

配置语法

  • 布尔类型Boolean
    • isFailed => true
  • 数值类型Number
    • port => 33
  • 字符串类型String
    • name => “Hello world”
  • 数组Array/List
    • user => [{id => 1, name => bob}, {id => 2, name => jane}]
  • 哈希类型Hash
  • 注释
  • 在配置中可以引用Logstash Event的属性(字段)，主要有如下两种方式：
    • 直接引用字段值
    • 在字符串中以sprintf方式引用
  • 支持条件判断语法，从而扩展了配置的多样性

1. input插件详解及glob讲解

stdin

    • tags类型为array，自定义该事件的tag，可用于后续判断
    • add_field类型为hash，为该事件添加字段
