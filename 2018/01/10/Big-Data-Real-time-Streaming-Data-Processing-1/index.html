初识实时流处理

1. 实时流处理产生背景

1. 业务现状分析

  • 需求：统计主站每个(指定)课程访问的客户端，地域信息分布情况
  • 实现步骤：
    • 课程编号，ip信息，useragent
    • 进行相应的统计分析操作：MapReduce/Spark
  • 项目架构
    • 日志收集：Flume
    • 离线分析：MapReduce/Spark
    • 统计结果图形化
  • 问题：小时级别？10分钟？5分钟？1分钟？秒级别？
  • 如何解决：实时流处理框架

2. 实时流处理产生背景

  • 时效性高
  • 数据量大

3. 实时流处理概述

  • 实时计算
    • 响应时间比较短，受到时间约束
    • 延迟性要求比较低
  • 流式计算
    • 在不断地产生的数据流上进行计算，数据是没有尽头的
  • 实时流式计算
    • 在不断产生的数据流上进行实时计算

4. 离线计算与实时计算对比

  • 数据来源
    • 离线：
      • HDFS历史数据
      • 数据量比较大
    • 实时：
      • 消息队列(Kafka)
      • 实时新增/修改过来的某一笔数据
  • 处理过程
    • 离线：MapReduce：Map + Reduce
    • 实时：Spark(DStream/SS)
  • 处理速度
    • 离线：慢
    • 实时：快
  • 进程
    • 离线：启动 + 销毁
    • 实时：7*24


2. 实时流处理框架

1. 实时流处理框架对比

  • Apache Storm
    • Apache Storm is a free and open source distributed realtime computation system.
  • Apache Spark Streaming
    • 微小批次处理框架(1秒1次，2秒1次)
  • IBM Stream
  • Yahoo!S4
  • LinkedIn Kafka
